\chapter{SAT y el problema del vacío para la intersección con un lenguaje regular}
\label{ch:el-teorema}

En esta sección se presenta el lenguaje $\satlang(F)$ al cual pertenecen todas las interpretaciones que satisfacen a una fórmula $F$ y se muestra una forma de construirlo a partir de la intersección del lenguaje $\repeatlang$ con un autómata finito determinista. Esta construcción permite resolver el problema de la satisfacibilidad booleana (SAT) para una fórmula $F$ a través de la decisión de si el lenguaje $\satlang(F)$ es vacío o no.

El capítulo se estructura de la siguiente forma: en la sección \ref{sec:codification} se muestra cómo codificar las interpretaciones de una fórmula usando el alfabeto $\{0, 1, d\}$ y se define el lenguaje $\satlang(F)$. En la sección \ref{sec:automata-cnf} se relaja el problema de satisfacibilidad booleana para permitir que cada cláusula de la fórmula se satisfaga por una interpretación distinta, y se construye un autómata que reconoce el lenguaje de las cadenas que representan a las interpretaciones que satisfacen a cada cláusula de la fórmula. Finalmente, en la sección \ref{sec:intersec-wplus} se muestra cómo obtener el lenguaje $\satlang(F)$ a partir de la intersección del lenguaje reconocido por dicho autómata con el lenguaje $\repeatlang$. Por último, se presenta por qué para todo formalismo que represente el lenguaje $\repeatlang$ decidir si es vacío o construir la intersección con un lenguaje regular son problemas NP-duros.

A continuación se presenta cómo codificar una interpretación de una fórmula booleana como una cadena sobre el alfabeto $\{0, 1, d\}$.

\section{Codificación de las interpretaciones de una fórmula booleana como cadenas}
\label{sec:codification}

Una formula booleana $F$, con $n$ variables $X = \{x_1, x_2, \ldots, x_n\}$ y $m$ cláusulas tiene la siguiente estructura:
$$
    F = C_1 \land C_2 \land \ldots \land C_m,
$$

donde cada cláusula $\Ci$ es una disyunción de literales
$$
    \Ci = t_{i1} \lor t_{i2} \lor \ldots \lor t_{ik_i}
$$

y cada literal $t_{ij}$ es una variable $x_l$ o su negación $\neg x_l$ para algún $l \in \{1, \ldots, n\}$.

El problema de satisfacibilidad booleana (SAT) consiste en decidir si existe una asignación de valores a las variables que haga verdadera a la fórmula.

Para representar las interpretaciones de $F$ como cadenas, se asignará a cada variable un símbolo binario: $0$ para falso y $1$ para verdadero. De esta forma, cada interpretación $v$ de las variables de $F$ se puede representar como una cadena de longitud $n$ sobre el alfabeto $\{0, 1\}$, donde el $i$-ésimo símbolo de la cadena representa el valor asignado a la variable $x_i$ por la interpretación $v$.

Por ejemplo, si $X = \{x_1, x_2, x_3\}$ y una interpretación $v$ asigna $v(x_1) = 0$, $v(x_2) = 1$ y $v(x_3) = 0$, entonces la cadena que representa a esta interpretación sería 010.

En este trabajo se propone resolver el problema de la satisfacibilidad booleana reduciéndolo a determinar si un lenguaje es vacío o no.

Para ello se codificarán las interpretaciones de la siguiente manera:
\begin{enumerate}
    \item A cada interpretación $v$ se le asigna una cadena $w$ de longitud $n$ sobre el alfabeto $\{0, 1\}$, donde el $i$-ésimo símbolo de la cadena representa el valor asignado a la variable $x_i$ por la interpretación.
    \item Se introduce un nuevo símbolo $d$ que se utilizará como separador entre las cláusulas de la fórmula.
    \item Se copia la cadena $w$ asociada a cada interpretación $v$ $m$ veces, una por cada cláusula de la fórmula, y se separan con el símbolo $d$.
\end{enumerate}

De esta forma, a cada interpretación $v$ de las variables de $F$ se le asocia una cadena de la forma $(wd)^m$, donde $w$ es la cadena binaria que representa a la interpretación $v$ y $d$ es el símbolo separador.

Por ejemplo, para la fórmula $F  = (x_1 \lor \neg x_3) \land (-x_1 \lor x_2)$, si una interpretación $v$ asigna $v(x_1) = 0$, $v(x_2) = 1$ y $v(x_3) = 0$, entonces la cadena que se le asociará a esta interpretación será $010d010d$.

En el resto del capítulo, dada una interpretación $v$ de una fórmula con $n$ variables y $m$ cláusulas, se dirá que la cadena $w \in \{0, 1\}^n$ codifica a $v$, o que representa la interpretación $v$, y se denominará representación extendida de $v$ a la cadena $(wd)^m$ asociada.

El lenguaje de todas las interpretaciones que satisfacen a una fórmula $F$ de $m$ cláusulas se puede definir como el siguiente lenguaje sobre el alfabeto $\{0, 1, d\}$:

$$
    \satlang(F) = \{(wd)^m \suchthat w \text{ codifica a una interpretación que satisface } F = C_1 \land \ldots \land C_m \}.
$$


Como para cada interpretación $v$ existe una y solo una cadena asociada en $\satlang(F)$, entonces el lenguaje $\satlang(F)$ es vacío si y solo si no existe una interpretación que haga verdadera a $F$.

Para decidir si existe una interpretación que haga verdadera a $F$ basta con decidir si el lenguaje $\satlang(F)$ es vacío o no. Por tanto, se puede reducir el problema SAT para una fórmula $F$ a decidir si el lenguaje $\satlang(F)$ es vacío o no.

En el resto del capítulo se describe como construir el lenguaje $\satlang(F)$. En la siguiente sección se presenta un autómata que reconoce el lenguaje de las cadenas que permiten que cada cláusula se satisfaga por una interpretación distinta. En la sección \ref{sec:intersec-wplus} se muestra como obtener el lenguaje $\satlang(F)$ a partir de la intersección del lenguaje reconocido por dicho autómata con el lenguaje $\repeatlang $.

\section{Autómata para el lenguaje $\indlang(F)$}
\label{sec:automata-cnf}

En esta sección se considera una versión relajada del problema, en la que se permite que cada cláusula de la fórmula se satisfaga por una interpretación distinta, en lugar de exigir una única interpretación que satisfaga a todas las cláusulas simultáneamente. Bajo esta relajación se define el lenguaje

$$
    \indlang(F) = \{ w_1 d \ldots  w_m d \suchthat w_i \text{ codifica una interpretación que satisface la cláusula } \Ci\}
$$

y se construye un autómata $\aut{F}$ que lo reconoce. Más adelante, en la sección \ref{sec:intersec-wplus}, al intersectar este lenguaje con $\repeatlang$ se recuperará la restricción de que todas las cláusulas compartan la misma interpretación.


Para construir dicho autómata se construirá primero un autómata $\automata_C$ para cada cláusula de la fórmula, y luego estos se concatenarán para obtener el autómata que reconoce el lenguaje $\indlang(F)$. A continuación se describe la construcción del autómata para cada cláusula de la fórmula.


\subsection{Automata cláusula}
En esta sección se construye el autómata $\automata_C$ que reconoce el lenguaje de las cadenas que representan a las interpretaciones que satisfacen a una cláusula $C$ de una fórmula $n$ variables $X= \{x_1, \ldots, x_n\}$. Se denotará a este lenguaje $\clauselang{C}$ y se define como sigue:
$$
    \clauselang{C} = \{ w \suchthat w \text{ representa a una interpretación que satisface la cláusula } C \}
$$

Para una fórmula con $n$ variables, los estados del autómata $\aut{C}$ son de la forma $(q_f, i)$ o $(q_t, i)$, donde $i \in \{0, \ldots, n\}$. Estos estados representan la información de si luego de leer los $i$ primeros símbolos de la cadena, ya se encontró un literal de la cláusula que evalúa verdadero o no.

\begin{itemize}
    \item Un estado de la forma $(q_f, i)$ representan que luego de leer $i$ símbolos de la cadena no se ha leído el valor de una variable $x_j$ con $1 \leq j \leq i$ cuyo literal asociado en la cláusula sea verdadero.
    \item Un estado de la forma $(q_t, i)$ representan que luego de leer $i$ símbolos de la cadena, ya se ha leído el valor de una variable $x_j$ con $1 \leq j \leq i$ cuyo literal asociado en la cláusula es verdadero.
\end{itemize}

Por ejemplo, un estado $(q_f, 2)$ representa que luego de leer los valores de las dos primeras variables de la interpretación, no se ha leído el valor de ninguna variable cuyo literal asociado en la cláusula evalúe verdadero bajo dicha interpretación. En cambio un estado $(q_t, 7)$ significa que luego de leer los valores de las primeras 7 variables de la interpretación, se ha leído el valor de una variable cuyo literal asociado en la cláusula evalúa verdadero bajo dicha interpretación.

El estado inicial del autómata será $(q_f, 0)$, pues al no haber leído ningún símbolo de la cadena, no se ha leído el valor asociado a ninguna variable y por tanto la cláusula no puede ser verdadera.

El estado de aceptación del autómata será $(q_t, n)$, y significa que luego de leer los $n$ símbolos de la cadena, se ha leído el valor de una variable cuyo literal asociado en la cláusula es verdadero, por lo tanto la cláusula es verdadera.

Las transiciones entre los estados solo existen entre estados de la forma $(q_f, i)$ o $(q_t, i)$ y estados de la forma $(q_f, i+1)$ o $(q_t, i+1)$.

\begin{itemize}
    \item Un estado de la forma $(q_t, i)$ representa que luego de leer $i$ símbolos de la cadena, se ha leído el valor de una variable cuyo literal asociado en la cláusula es verdadero, por lo tanto, sin importar los símbolos restantes, la cláusula seguirá siendo verdadera. Luego, se tiene la transición
          $$
              \delta((q_t, i), a) = (q_t, i+1) \text{ para cualquier símbolo } a \in \binalph.
          $$
    \item Por otro lado un estado de la forma $(q_f, i)$ representa que luego de leer $i$ símbolos de la cadena, no se ha leído el valor de una variable cuyo literal asociado en la cláusula sea verdadero, por lo tanto, para que la cláusula sea verdadera, el siguiente símbolo leído debe ser tal que el literal asociado a la variable $x_{i+1}$ en la cláusula evalúe verdadero. Si la variable $x_{i+1}$ no aparece en la cláusula, o aparece pero el símbolo leído hace que el literal asociado evalúe falso, entonces la cláusula seguirá siendo falsa. Por lo tanto se tiene la transición
          $$
              \delta((q_f, i), a) = \begin{cases}
                  (q_t, i+1) & \text{si } (x_{i+1} \in C \land a = 1) \lor (\neg x_{i+1} \in C \land a = 0) \\
                  (q_f, i+1) & \text{en otro caso}
              \end{cases}
          $$
\end{itemize}

A continuación se define formalmente el autómata cláusula.

\begin{definition}
    Para una cláusula $C$ de una fórmula $F$ con $n$ variables, se define el autómata cláusula $\automata_C = (Q, \binalph, \delta, (q_f, 0), F)$ donde:
    \begin{itemize}
        \item $Q = \{(q_f, i), (q_t, i) \suchthat i \in \{0, \ldots, n\}\}$
              % el autómata (q_t, 0) es verdad que no tiene sentido que exista en la práctica, pero es más fácil definir el autómata así y ayuda a que no se tenga que considerar ese caso especial en las demostraciones prefiero tenerlo ahí si no es de vida o muerte :D
        \item $(q_f, 0)$ es el estado inicial
        \item $\delta$ es la función de transición definida para todo $i \in \{0, \ldots, n-1\}$ y $a \in \{0, 1\}$ por:
              $$
                  \begin{aligned}
                      \delta((q_t, i), a)
                       & = (q_t, i+1), \\
                      \delta((q_f, i), a)
                       & =
                      \begin{cases}
                          (q_t, i+1)
                           & \text{si } (x_{i+1} \in C \land a = 1)
                          \lor (\neg x_{i+1} \in C \land a = 0),    \\
                          (q_f, i+1)
                           & \text{en otro caso}.
                      \end{cases}
                  \end{aligned}
              $$
        \item $F = \{(q_t, n)\}$
    \end{itemize}

\end{definition}

Para una fórmula con $n$ variables, el autómata $\automata_C$ tendrá $2(n + 1) = O(n)$ estados: $n+1$ de la forma $(q_f, i)$ y $n+1$ de la forma $(q_t, i)$, con $i \in \{0, \ldots, n\}$.

A continuación se demostrará que el lenguaje que reconoce el autómata $\aut{C}$ es exactamente el lenguaje
$$
    \clauselang{C} = \{ w \suchthat w \text{ codifica a una interpretación que satisface la cláusula } C \}.
$$
Para ello, primero se demostrará el siguiente lema que describe la propiedad mantenida por el autómata luego de leer $k$ variables.

\begin{lemma}
    \label{lem:ac}
    Dado un autómata cláusula $\aut{C}$ y una cadena de entrada $w = w_1, w_2 \dots w_r$, luego de leer $k \leq r$ símbolos el autómata $\aut{C}$ se encuentra en un estado verdadero $(q_t, k)$ si y solo si existe una variable $x_i$ de $C$ con $1 \leq i \leq k$  tal que $w_i = 1 \land x_i \in C$ o $w_i = 0 \land \neg x_i \in C$.
\end{lemma}

\paragraph{Demostración} Primero se demostrará que si luego de leer $k$ símbolos el autómata $\automata_C$ se encuentra en un estado verdadero entonces existe una variable $x_i$ de $C$ tal que el literal asociado en $C$ es verdadero. Formalmente:

\begin{equation}
    \label{eq:ac-implies}
    \extran((q_f, 0), a_1 \ldots a_k) = (q_t, k) \implies \exists i \in \{1, \ldots, k \} \text{ con } (a_i = 1 \land x_i \in C) \lor (a_i = 0 \land \neg x_i \in C).
\end{equation}


Se demostrará por inducción sobre $k$.

\subparagraph{Caso base} Para $k = 1$ si se tiene que $\delta((q_f, 0), a_1) = (q_t, 1)$, entonces por la definición de la función de transición se tiene que $a_1$ es tal que el literal asociado a la variable $x_1$ en $C$ es verdadero. Por lo tanto, se cumple la afirmación.

\subparagraph{Paso inductivo} Supóngase que la afirmación se cumple para $k$. Supóngase además que luego de leer $k+1$ símbolos el autómata se encuentra en un estado verdadero, formalmente esto es $\extran((q_f, 0), a_1 \ldots a_{k+1}) = (q_t, k+1)$. Se tienen dos casos:

\begin{itemize}
    \item $\extran((q_f, 0), a_1 \ldots a_k) = (q_t, k)$, entonces por la hipótesis inductiva existe una variable $x_i$ de $C$ con $i \leq k$ tal que el literal asociado en $C$ es verdadero, por lo tanto se cumple la afirmación.
    \item $\extran((q_f, 0), a_1 \ldots a_k) = (q_f, k)$, entonces por la definición de la función de transición se tiene que $a_{k+1}$ es tal que el literal asociado a la variable $x_{k+1}$ en $C$ es verdadero. Por lo tanto, se cumple la afirmación.

\end{itemize}

Por tanto para todo $k \in \{1, \ldots, n\}$ se cumple que si el autómata luego de leer $k$ símbolos se encuentra en un estado verdadero, entonces existe una variable $x_i$ de $C$ con $i \leq k$ tal que el literal asociado en $C$ es verdadero.

A continuación se demuestra la implicación recíproca: si existe una variable $x_i$ de $C$ con $i \leq k$ tal que el literal asociado en $C$ es verdadero, entonces el autómata luego de leer $k$ símbolos se encuentra en un estado verdadero. Formalmente:

\begin{equation}
    \label{eq:ac-implied-by}
    \exists i \in \{1, \ldots, k \} \text{ con } (a_i = 1 \land x_i \in C) \lor (a_i = 0 \land \neg x_i \in C) \implies \extran((q_f, 0), a_1 \ldots a_k) = (q_t, k).
\end{equation}


Se demostrará por inducción sobre $k$.

\subparagraph{Caso base} Para $k = 1$ se cumple que $x_1 \in C$ y $a_1 = 1$ o $\neg x_1 \in C$ y $a_1 = 0$, entonces por la definición de la función de transición se tiene que $\delta((q_f, 0), a_1) = (q_t, 1)$, por lo tanto $\extran((q_f, 0), a_1) = (q_t, 1)$ y se cumple la afirmación.

\subparagraph{Paso inductivo} Supóngase que la afirmación se cumple para $k$: existe una variable $x_i$ de $C$ con $i \leq k+1$ tal que el literal asociado en $C$ es verdadero.

Se tienen dos casos: existe un literal $x_j$ con $j \leq k$ tal que el literal asociado a la variable $x_j$ en $C$ es verdadero, o solo el literal asociado a la variable $x_{k+1}$ en $C$ es verdadero.

\begin{itemize}
    \item Primer caso. Existe $i \leq k$ tal que el literal asociado a la variable $x_i$ en $C$ es verdadero, entonces por hipótesis inductiva,
          $$
              \extran((q_f, 0), a_1 \ldots a_k) = (q_t, k).
          $$
          Como $\extran ((q_t, k), a_{k+1}) = (q_t, k+1)$, para todo $a_{k+1} \in \binalph$ concluye que
          $$
              \extran((q_f, 0), a_1 \ldots a_{k+1}) = (q_t, k+1),
          $$
          y por lo tanto se cumple la afirmación.
    \item En el segundo caso, solo el literal asociado a $x_{k+1}$ en $C$ es verdadero. Esto significa que:
          $$
              (a_{k+1}=1 \land x_{k+1}\in C)
              \lor
              (a_{k+1}=0 \land \neg x_{k+1}\in C).
          $$
          Entonces, por definición de \(\delta\) se tiene que
          $$
              \delta((q_f,k),a_{k+1})=(q_t,k+1)
          $$
          Como para todo $i \leq k$ el literal asociado a $x_i$ en $C$ es falso, por la implicación \ref{eq:ac-implies} ya demostrada, se tiene que
          $$
              \extran((q_f, 0), a_1 \ldots a_k) \neq (q_t, k).
          $$
          Dado que luego de leer $k$ símbolos el autómata no se encuentra en un estado verdadero, se tiene que
          $$
              \extran((q_f, 0), a_1 \ldots a_k) = (q_f, k).
          $$


          Por tanto,
          $$
              \extran((q_f, 0),a_1\ldots a_{k+1})=(q_t,k+1).
          $$
          Con esto se cumple la afirmación.
\end{itemize}

Luego de demostrar ambas implicaciones se concluye que para todo $k \in \{1, \ldots, n\}$ se cumple que luego de leer $k$ símbolos el autómata se encuentra en un estado verdadero si y solo si existe una variable $x_i$ de $C$ con $i \leq k$ tal que el literal asociado en $C$ es verdadero. \qed


Ahora se procederá a enunciar y demostrar el teorema que establece la equivalencia entre el lenguaje reconocido por el autómata $\aut{C}$ y el lenguaje $L_C$.

\begin{theorem}
    \label{thm:cautomata}
    Sean $C$ una cláusula de una fórmula con variables $X = \{x_1, \ldots, x_n\}$, $\aut{C}$ el autómata cláusula asociado a $C$ y $L_C$ el lenguaje de las cadenas que representan a las interpretaciones que satisfacen a $C$. Se cumple que $L_C = L(\aut{C})$.
\end{theorem}

\paragraph{Demostración}
Para demostrar que $\clauselang{C} = L(\aut{C})$ se deben demostrar dos inclusiones: primero se demostrará que $\clauselang{C} \subseteq L(\aut{C})$ y luego que $L(\aut{C}) \subseteq \clauselang{C}$.

\subparagraph{$\clauselang{C} \subseteq L(\aut{C})$:}

\hfill

Supóngase que $w \in \clauselang{C}$, entonces $w = a_{1} \ldots a_{n}$ representa una interpretación $v$ tal que $(C)_v = 1$, por tanto existe un literal $t$ de $C$ tal que $(t)_v = 1$. Por el lema \ref{lem:ac} con $k = n$ se cumple que $\extran((q_f, 0), w) = (q_t, n)$, luego $w \in L(\aut{C})$.

\subparagraph{$L(\aut{C}) \subseteq \clauselang{C}$:}

\hfill

Supóngase que $w \in L(\aut{C})$, entonces $\extran((q_f, 0), w) = (q_t, n)$, por el lema \ref{lem:ac} con $k = n$ se cumple que existe un literal $t$ de $C$ tal que $(t)_v = 1$, luego $(C)_v = 1$ y $w$ representa a una interpretación $v$ tal que $(C)_v = 1$, por tanto $w \in \clauselang{C}$.

Como se tiene que $\clauselang{C} \subseteq L(\aut{C}) \text{ y } L(\aut{C}) \subseteq \clauselang{C}$, entonces

$$
    \clauselang{C} = L(\aut{C}).
$$
\qed

A continuación se describirá la construcción del autómata que reconoce el lenguaje $L_F$ a partir de los autómatas $\aut{C}$ asociados a cada cláusula de la fórmula $F$.

\subsection{Autómata fórmula}

Dada la fórmula $F$ con cláusulas $C_1, \ldots, C_m$, se tienen autómatas $\aut{\Ci}$ que reconocen el lenguaje $\clauselang{C_i}$ para cada cláusula $C_i$ de la fórmula, bajo la suposición en la que se permite que cada cláusula se satisfaga por una interpretación distinta. Para construir el autómata que reconoce el lenguaje

$$
    \indlang(F) = \{ w_1 d \ldots  w_m d \suchthat w_i \text{ codifica una interpretación que satisface la cláusula } \Ci\}.
$$

se necesita concatenar los autómatas $\aut{\Ci}$ para cada cláusula $C_i$ de la fórmula, pero antes es necesario modificar estos autómatas para que reconozcan las cadenas que representan a una interpretación que satisface a la cláusula con el símbolo separador $d$ al final. De esta forma, para cada cláusula $C_i$ el autómata resultante reconocerá las cadenas de la forma $w_id$, donde $w_i$ representa a una interpretación que satisface a la cláusula $C_i$.

Primero se construye un autómata que reconozca el lenguaje $\clauselang{C}$ con los separadores de cláusulas: el lenguaje $L_{Cd} = \{(wd) \suchthat w \in \clauselang{C}\}$. Para esto basta con modificar el autómata $\aut{C}$ para que si está en el estado de aceptación reconozca la cadena $d$ después de haber leído una cadena de $\clauselang{C}$.

A continuación se define formalmente el autómata cláusula con separador asociado a una cláusula $C_i$ de la fórmula $F$.

\begin{definition}
    Dada una cláusula $C_i$ de una fórmula con las variables $X=\{x_1, \ldots x_n\}$ sea
    $$
        \aut{\Ci} = (Q, \binalph, \delta, (q_f, 0), \{(q_t, n)\})
    $$
    el autómata cláusula asociado a $C_i$. Se define el autómata cláusula con separador asociado a $C_i$ como
    $$
        \daut{Ci} = (Q \cup \{q_d\}, \{0, 1, d\}, \delta', (q_f, 0), \{q_d\})
    $$
    donde $\delta'$ se define como
    $$
        \delta'(q, a) = \begin{cases}
            \delta(q, a) & \text{si } q \in Q \text{ y } q \neq (q_t, n) \\
            q_d          & \text{si } q = (q_t, n) \text{ y } a = d
        \end{cases}
    $$
\end{definition}

El autómata $\daut{Ci}$ reconoce el lenguaje de las cadenas que representan a las interpretaciones que satisfacen a la cláusula $C_i$ con el símbolo separador $d$ al final. En la figura \ref{fig:clause_automaton_d} de la página \pageref{fig:clause_automaton_d} se muestra el autómata cláusula con separador asociado a la cláusula $C = x_1 \lor \neg x_2$ de una fórmula con variables $X = \{x_1, x_2, x_3\}$.

Los autómatas $\daut{Ci}$ para cada cláusula $C_i$ reconocen el lenguaje de las cadenas que representan a las interpretaciones que satisfacen a la cláusula $C_i$ con el símbolo separador $d$ al final. Formalmente se tiene el siguiente teorema.

\begin{theorem}
    \label{thm:dautomata}
    Sea una cláusula $C_i$ de una fórmula con variables $X = \{x_1, \ldots, x_n\}$, $\daut{Ci}$ el autómata cláusula con separador asociado a $C_i$ y el lenguaje $\clauselang{C_i}$ de las cadenas que representan a las interpretaciones que satisfacen a la cláusula $C_i$.
    Se cumple que
    $$
        L(\daut{Ci}) = \{(wd) \suchthat w \in \clauselang{C_i}\}.
    $$
\end{theorem}

La demostración es directa dada la definición de $\delta'$ y el hecho de que $L_{C_i} = L(\aut{C_i})$. \qed


El siguiente paso es concatenar los autómatas cláusula con separador para construir un autómata que reconozca el lenguaje $\indlang(F)$.

Para ello, se concatenan los autómatas $\daut{\Ci}$ para cada cláusula $C_i$ de la fórmula mediante el procedimiento de concatenación de \cite{automataTheory}: se conecta el estado de aceptación $q_{di}$ del autómata $\daut{\Ci}$ con el estado inicial $(q_{f(i+1)}, 0)$ del autómata $\daut{C_{i+1}}$ para cada $i < m$, mediante la transición
$$
    \delta((q_{ti}, n), \eword) = (q_{f(i+1)}, 0).
$$

Estas transiciones introducen indeterminismo en el autómata. Sin embargo, existe una construcción estándar para convertir un autómata no determinista a un autómata determinista \cite{automataTheory}. Esta construcción puede tomar un tiempo exponencial en el número de estados del autómata no determinista. No obstante, en este caso particular la construcción es inmediata, pues las transiciones $\eword$ solo conectan el estado final de un autómata con el estado inicial del siguiente. Esto permite que el indeterminismo pueda ser eliminado sustituyendo ambos estados por uno nuevo y modificando la función de transición.

A continuación se define formalmente el autómata fórmula asociado a una fórmula $F$ con cláusulas $C_1, \ldots, C_m$.

\begin{definition}
    Sean una fórmula $F = C_1 \land \ldots \land C_m$ con variables $X = \{x_1, \ldots, x_n\}$. Sea
    $$
        \daut{Ci} = (Q_i, \{0, 1, d\}, \delta_i, (q_{fi}, 0), (q_{di}))
    $$
    el autómata cláusula con separador asociado a cada cláusula $C_i$ de la fórmula $F$.

    Se define el autómata fórmula $\aut{F}$ asociado a $F$ como el autómata finito determinista resultante de eliminar el indeterminismo del autómata
    $$
        \mathcal{N}_F = (Q, \{0, 1, d\}, \delta, (q_{f1}, 0), \{q_{dm}\})
    $$ donde:
    \begin{itemize}
        \item $Q = \bigcup_{i=1}^{m} Q_i$
        \item La función de transición $\delta$ es la unión de las funciones de transición $\delta_i$ y las transiciones $\delta((q_{di}, n), \eword) = (q_{f(i+1)}, 0)$ para cada $i < m$.
    \end{itemize}
\end{definition}

Con la construcción del autómata fórmula se ha demostrado el siguiente teorema.

\begin{theorem}
    El autómata $\automata_F$ reconoce el lenguaje $\indlang(F)$ definido como
    $$
        \indlang(F) = \{ w_1 d \ldots w_m d \suchthat w_i \text{ codifica una interpretación que satisface la cláusula } C_i\}.
    $$
\end{theorem}

Dado que los lenguajes regulares son aquellos que pueden ser reconocidos por un autómata finito \cite{automataTheory}, se tiene como resultado que el lenguaje $\indlang(F)$ es un lenguaje regular.

\begin{corollary}
    El lenguaje $\indlang(F)$ es un lenguaje regular.
\end{corollary}

Las cadenas del lenguaje $\indlang(F)$ no necesariamente corresponden a la representación extendida de una interpretación que satisface a la fórmula $F$, ya que cada cláusula de la fórmula se puede satisfacer por una interpretación diferente.

Por ejemplo, dada la fórmula $F = (x_1 \lor \neg x_2) \land (\neg x_1 \lor x_3)$, la cadena $101d001d$ pertenece a $\indlang(F)$ pero no representa una interpretación $F$. Esto se debe a que $101$ y $001$ son cadenas que representan dos interpretaciones diferentes que satisfacen a cada cláusula de la fórmula $F$.

En la siguiente sección se demostrará como obtener el lenguaje $\satlang(F)$ de todas las interpretaciones que satisfacen a la fórmula $F$ a partir de intersectar el lenguaje $\indlang(F)$ con el lenguaje $\repeatlang$ que obliga a que las interpretaciones sean las mismas para todas las cláusulas de la fórmula.

\section{$\satlang(F)$ a partir de la intersección con un lenguaje regular}
\label{sec:intersec-wplus}

En esta sección se demostrará que el lenguaje $\satlang(F)$ de todas las interpretaciones que satisfacen a la fórmula $F$ se puede obtener a partir de la intersección del lenguaje $\indlang(F)$ con el lenguaje $\repeatlang$.

Dada una fórmula en forma normal conjuntiva
$$
    F = C_1 \land C_2 \land \ldots \land C_m,
$$
el lenguaje $\indlang(F)$ es el lenguaje de las cadenas $w_1d w_2d \ldots d w_md$, donde $w_i$ representa a una interpretación $v_i$ que satisface a la cláusula $C_i$. Dado que es posible que existan $i, j \leq m$ tales que $w_i \neq w_j$, las cadenas de $\indlang(F)$ no necesariamente están asociadas a una interpretación que satisface la formula $F$.

Para obtener el lenguaje $\satlang(F)$ de todas las interpretaciones que satisfacen a la fórmula $F$, es necesario imponer la restricción de que las interpretaciones $v_i$ asociadas a cada cláusula $C_i$ sean iguales. Para ello se intersectará el lenguaje $\indlang(F)$ con el lenguaje $\repeatlang$. Este lenguaje se define a continuación.

\begin{definition}
    El lenguaje $\repeatlang$ se define como:
    $$
        \repeatlang = \{ (wd)^k \suchthat w \in \binalph^*, k \geq 1 \}
    $$

    $\repeatlang$ contiene cadenas sobre el alfabeto $\{0, 1, d\}$ que representan una cadena binaria concatenada con el símbolo $d$ repetida una o más veces.
\end{definition}



El lenguaje $\repeatlang$ obliga a que las cadenas que representan a las interpretaciones asociadas a cada cláusula sean iguales.

Por ejemplo, las cadenas $101d101d101d$, $101d101d$ y $101d$ pertenecen a $\repeatlang$, mientras que las cadenas $101d100d$ y $101d100d101d$ no pertenecen a $\repeatlang$.

A continuación se demuestra que el lenguaje $\indlang(F) \cap \repeatlang$ es exactamente el lenguaje $\satlang(F)$ de todas las interpretaciones que hacen verdadera a $F$.

\begin{theorem}
    \label{thm:lv-intersection}
    Para toda fórmula $F$ con $n$ variables y $m$ cláusulas, sean:
    \begin{itemize}
        \item $\satlang(F)$ es el lenguaje de todas las interpretaciones que satisfacen a la fórmula $F$,
        \item $\indlang(F)$ es el lenguaje de las cadenas $w_1d w_2d \ldots d w_md$, donde $w_i$ representa a una interpretación $v_i$ que satisface a la cláusula $C_i$, y
        \item $\repeatlang$ es el lenguaje de las cadenas de la forma $(wd)^k$ con $w \in \binalph^*$ y $k \geq 1$.
    \end{itemize}
    Entonces se cumple que $\satlang(F) = \indlang(F) \cap \repeatlang$
\end{theorem}

\paragraph{Demostración}

Para demostrar que $\satlang(F) = \indlang(F) \cap \repeatlang$ se demostrará primero que $\satlang(F) \subseteq \indlang(F) \cap \repeatlang$ y luego que $\indlang(F) \cap \repeatlang \subseteq \satlang(F)$.

\subparagraph{$\satlang(F) \subseteq \indlang(F) \cap \repeatlang$:}

\hfill

Sea $\alpha \in \satlang(F)$, por definición de $\satlang(F)$, se tiene que $\alpha$ es una cadena de la forma $(wd)^m$ donde $w$ representa a una interpretación $v$ que satisface a la fórmula $F$.

Se tiene que $\alpha \in \repeatlang$, ya que $\alpha$ es una cadena de la forma $(wd)^m$ con $w \in \binalph^*$ y $m \geq 1$.

Por otro lado, como $v$ satisface a la fórmula $F$, entonces $v$ satisface todas las cláusulas de $F$. Por lo tanto, $\alpha = w_1d \ldots w_md$ donde $w_i = w$ representa a la interpretación $v$ que satisface a la cláusula $C_i$ para todo $i \leq m$, por lo tanto $\alpha \in \indlang(F)$.

Al $\alpha$ pertenecer a ambos conjuntos se concluye que $\alpha \in \indlang(F) \cap \repeatlang$, luego $\satlang(F) \subseteq \indlang(F) \cap \repeatlang$.


\subparagraph{$\indlang(F) \cap \repeatlang \subseteq \satlang(F)$:}

\hfill

Sea $\alpha \in \indlang(F) \cap \repeatlang$. Como $\alpha \in \repeatlang$, entonces existen $w \in \binalph^*$ y $k \geq 1$ tales que
\begin{equation}
    \alpha = (wd)^k.
    \label{eq:alpha-l01d}
\end{equation}

Como $\alpha \in \indlang(F)$, entonces
\begin{equation}
    \label{eq:alpha-lf}
    \alpha = w_1 d w_2 d \ldots d w_m d
\end{equation}

donde $w_i$ representa a una interpretación $v_i$ que satisface a la cláusula $C_i$ para todo $i \leq m$. Juntando las ecuaciones \ref{eq:alpha-l01d} y \ref{eq:alpha-lf} se tiene
\begin{equation}
    w_1 d w_2 d \ldots d w_m d = (wd)^k.
\end{equation}

Como en el lado izquierdo de \ref{eq:alpha-lf} el símbolo separador aparece exactamente $m$ veces, entonces $k = m$. Luego,
\begin{equation}
    w_1 d w_2 d \ldots d w_m d = (wd)^m.
\end{equation}

Por tanto las cadenas separadas por el símbolo $d$ son iguales a $w$, esto es, $$
    w_1 = w_2 = \ldots = w_m = w,
$$

al ser $w$ la cadena que representa a una interpretación $v$, entonces $w_i$ representa a la misma interpretación $v$ para todo $i \leq m$.

Por lo tanto, existe una interpretación $v$ tal que $v_i = v$ para todo $i \leq m$ que satisface todas las cláusulas de la fórmula, y por tanto satisface la fórmula. Luego $\alpha = (wd)^m \in \satlang(F)$ con $(F)_v = 1$, por tanto $\alpha \in \satlang(F)$. Luego $\indlang(F) \cap \repeatlang \subseteq \satlang(F)$.

Como se ha demostrado que $\satlang(F) \subseteq \indlang(F) \cap \repeatlang$ y que $\indlang(F) \cap \repeatlang \subseteq \satlang(F)$, se concluye que $\satlang(F) = \indlang(F) \cap \repeatlang$. \qed

Una consecuencia directa del teorema \ref{thm:lv-intersection} es el siguiente resultado.

\begin{theorem}
    \label{thm:el-teorema}
    Para cualquier formalismo G que genere el lenguaje $\repeatlang$ y sea cerrado bajo intersección con lenguajes regulares, la construcción de la intersección con lenguajes regulares o el problema de vacío es NP-duro.
\end{theorem}

\subparagraph{Demostración} Supóngase que existe un formalismo $G$ que genera el lenguaje $\repeatlang$ y es cerrado bajo intersección con lenguajes regulares. Se demostrará que el problema de decidir si la intersección de un lenguaje generado por dicho formalismo con un lenguaje regular es vacía es NP-duro. Para ello, se reducirá el problema SAT a dicho problema.

Sea $F$ una fórmula booleana con $n$ variables y $m$ cláusulas.

Se construye el autómata $\aut{F}$ que reconoce el lenguaje $\indlang(F)$. Luego se construye el lenguaje $\repeatlang$ a partir del formalismo $G$. Dado que $G$ es cerrado bajo intersección con lenguajes regulares, entonces se puede construir el lenguaje $\indlang(F) \cap \repeatlang$ a partir de la intersección de $\indlang(F)$ y $\repeatlang$.

Por el teorema \ref{thm:lv-intersection} se tiene que $\satlang(F) = \indlang(F) \cap \repeatlang$, por lo tanto decidir si existe una interpretación que hace verdadera a $F$ es equivalente a decidir si el lenguaje $\indlang(F) \cap \repeatlang$ es vacío o no.

La construcción del autómata $\aut{F}$ es polinomial en el tamaño de la fórmula $F$, ya que $\aut{F}$ tiene $O(mn)$ estados, donde $m$ es la cantidad de cláusulas y $n$ es el número de variables de $F$. Por tanto, SAT se reduce al problema de obtener la intersección de un lenguaje regular con un lenguaje generado por el formalismo $G$ y decidir si el resultado es vacío.

En consecuencia, al menos uno de estos problemas debe ser NP-duro:
\begin{itemize}
    \item construir la intersección con un lenguaje regular, o
    \item decidir si el lenguaje resultante es vacío.
\end{itemize}

Luego, para cualquier formalismo que genere el lenguaje $\repeatlang$ y sea cerrado bajo intersección con lenguajes regulares, la construcción de dicha intersección o el problema del vacío asociado son NP-duros. \qed

Una restricción importante que debe tener el formalismo $G$ para que el teorema \ref{thm:el-teorema} se cumpla que es la representación de $\repeatlang$ como lenguaje generado por $G$ sea $O(1)$. Puesto que el hecho de que la reducción anterior sea polinomial depende de que el tamaño de la representación del lenguaje $\repeatlang$ a partir del formalismo $G$ no dependa de la fórmula booleana $F$.

En este trabajo, al igual que en \cite{RCGSAT}, se conjetura que el lenguaje $\repeatlang$, tiene tamaño $O(1)$ en cualquiera de sus representaciones.

En la literatura consultada para este trabajo aparecen tres formalismos capaces de generar el lenguaje $\repeatlang$: las gramáticas de índice global \cite{globalIndexLanguages}, las gramáticas de concatenación de rango \cite{propertiesRCGBib} y las gramáticas libres de contexto múltiple paralelas (pMCFG) \cite{Seki1991OnMC}. En todos los casos la gramática que genera el lenguaje $\repeatlang$ tiene tamaño $O(1)$.


En el caso de las gramáticas de índice global, aplicando el teorema \ref{thm:el-teorema}, el problema de intersección con lenguajes regulares es NP-duro, aun cuando en \cite{globalIndexLanguages} se sugiere que este problema pueda ser resuelto en tiempo polinomial. Por otro lado, las gramáticas de concatenación de rango son cerradas bajo intersección con lenguajes regulares, pero el problema del vacío es indecidible, a menos que se cumplan ciertas restricciones, en cuyo caso no generan el lenguaje $\repeatlang$ \cite{propertiesRCGBib}.

De acuerdo con la literatura consultada para este trabajo, las gramáticas libres de contexto múltiple paralelas satisfacen simultáneamente todas las propiedades requeridas: generan el lenguaje $\repeatlang$ con representaciones de tamaño constante, son cerradas bajo intersección con lenguajes regulares y poseen problema del vacío decidible en tiempo polinomial. Teniéndose estas propiedades, de acuerdo con el teorema \ref{thm:el-teorema}, sería posible resolver SAT en tiempo polinomial, representando $\repeatlang$ con dicho formalismo.

Desafortunadamente, el procedimiento para determinar la intersección de una pMCFG con un lenguaje regular reportado por la literatura es incorrecto. En \cite{Seki1991OnMC} se propone un algoritmo para resolver el problema para las gramáticas libres de contexto múltiple (MCFG), un caso particular de las pMCFG, y propone que este algoritmo pueda ser utilizado para las pMCFG. Sin embargo como se muestra en el próximo capítulo mediante un contraejemplo, dicho algoritmo no es correcto para las pMCFG, y, como se demuestra en el capítulo \ref{ch:w-plus} determinar la intersección de una pMCFG con un lenguaje regular resulta ser un problema NP-duro.
